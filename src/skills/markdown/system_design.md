# System Design

## When to activate
- Defining architecture, modules, and integration boundaries.

## Workflow
1. Identify domain boundaries.
2. Define API/data contracts.
3. Specify deployment/runtime assumptions.

## Rules
- Keep architecture implementation-ready.
- Explicitly state trade-offs and risks.

## Output format
- Return architecture files and structured decisions.

## Failure conditions
- Missing data-flow boundaries.
- Ambiguous component ownership.
