# Testing

## When to activate
- Verifying generated backend/frontend code quality and behavior.

## Workflow
1. Define test strategy from requirements.
2. Add smoke and regression tests.
3. Report exact failures and impact.

## Rules
- Include deterministic assertions.
- Prefer minimal reproducible failing cases.

## Output format
- Return test files and prioritized issues.

## Failure conditions
- No test coverage for main user flows.
- Flaky or non-deterministic tests.
