# Product Management

## When to activate
- Turning strategy into implementable PRD and acceptance criteria.

## Workflow
1. Define user stories.
2. Define acceptance tests.
3. Rank scope by MVP priority.

## Rules
- Keep requirements testable.
- Explicitly call out non-goals.

## Output format
- Return PRD file + acceptance checklist.

## Failure conditions
- Ambiguous or non-testable requirements.
