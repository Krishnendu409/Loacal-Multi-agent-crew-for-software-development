# Backend Engineering

## When to activate
- Building APIs, service logic, persistence, and backend architecture.

## Workflow
1. Define contracts first.
2. Implement minimal working endpoints.
3. Add validation and error handling.
4. Add tests for happy path and failure path.

## Rules
- Keep modules cohesive.
- Use deterministic behavior.
- Avoid hidden global state.

## Output format
- Return files list with concrete paths and full content.

## Failure conditions
- Missing tests for critical paths.
- Unvalidated user inputs.
